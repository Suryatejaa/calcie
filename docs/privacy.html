CCALCIE

Privacy

Local-first by default

CALCIE is designed so personal context, screen captures, local databases, and profile imports stay on your machine unless you explicitly enable a cloud feature.

Local data

  • .calcie/ contains runtime state, vision captures, OCR, screen memory, and profile import backups.
  • calcie_profile.local.json stores imported user profile context.
  • calcie_history.db stores local chat history.

Release safety

The release hygiene check blocks private files like .env, .calcie, OCR dumps, screenshots, local DBs, and profile imports from shipping.

Cloud services

The backend is used for optional device sync, update manifests, feedback, and crash reports. Raw screen memory and imported ChatGPT memory are not uploaded by default.
